# قرارات العميل المطلوب حسمها (Client Decisions Required)
### جمعية خواطر أحلى شباب — اجتماع مراجعة المنتج

---

يرجى مراجعة النقاط والقرارات التالية مع إدارة الجمعية للحسم واكتفاء متطلبات مرحلة الإنتاج الحقيقي:

### 1. مسميات شريط التنقل السفلي (Bottom Navigation Labels)
- [ ] **الخيار أ (الموصى به)**: "الأسر" — للتعبير عن كفالة الأسر المتعففة وتفادي التكرار مع "الحالات العاجلة".
- [ ] **الخيار ب**: "الحالات" — المسمى العام المعتاد.

### 2. زر التبرع المركزي (Central Donation CTA)
- [ ] **الخيار أ (المعتمد في العرض)**: النص "تبرع" مع أيقونة القلب البارزة في منتصف الشريط.
- [ ] **الخيار ب**: النص "اتبرع".

### 3. القائمة الرسمية المعتمدة للمحافظات (Official Governorates List)
- [ ] الاعتماد النهائي لقائمة المحافظات الـ 12 الميدانية الحالية (القاهرة، الجيزة، الإسكندرية، الشرقية، المنوفية، المنيا، أسيوط، سوهاج، قنا، أسوان، الفيوم، شمال سيناء).
- [ ] تحديد الخدمات والمبادرات الخاصة بكل محافظة بشكل رسمي لمزامنتها مع قاعدة البيانات.

### 4. سياسة المفضلة والتصفح كزائر (Favorites & Guest Policy)
- [ ] هل تتطلب ميزة "المفضلة" تسجيل دخول رسمي لحفظ الحالات في حساب المستخدم، أم يُسمح بحفظها محلياً على الجهاز للزائر؟

### 5. التبرع السريع للزائر (Guest Donation Email Requirement)
- [ ] هل يلزم إدخال البريد الإلكتروني للزائر عند التبرع لإرسال إيصال التبرع الإلكتروني؟ أم يُكتفى برقم الهاتف أو التبرع المجهول؟

### 6. صلاحيات وطريقة وصول المستشارين للوحة التحكم (Consultant Portal Access)
- [ ] تحديد آلية تسجيل دخول مقدم الخدمة (هل يتم إنشاء حساب المستشار بواسطة مدير النظام فقط، أم عبر رابط مخصص برقم سري؟).
- [ ] تحديد ما إذا كانت الاستشارات تتم عبر زووم / تيمز، أم عبر مكالمات هاتفية وواتساب مباشرة.

### 7. دورة حياة حالة طلب الاستشارة (Consultation Status Workflow)
- [ ] اعتماد حالات الطلب الرسمية:
  1. `جديد` (عند التقديم).
  2. `قيد المراجعة` (من قبل فريق الفرز).
  3. `تم تحديد موعد` (بعد الاتفاق مع المستشار والمتقدم).
  4. `مكتمل` (بعد انتهاء الجلسة).
  5. `ملغي` (في حال تعذر التواصل أو الإلغاء).

### 8. إنشاء حساب تلقائي عند تقديم الاستشارة (Auto Account Creation)
- [ ] هل يؤدي تقديم الزائر لطلب استشارة إلى إنشاء حساب رسمي موثق على السيرفر تلقائياً بالبريد الإلكتروني، أم يظل الطلب مرتبطاً بالبريد فقط حتى يقوم بتأكيد بريده عبر OTP؟

### 9. موفر خدمة OTP للبريد الإلكتروني (Production Email Service Provider)
- [ ] اختيار موفر الخدمة المفضل لإرسال بريد رمز التحقق (مثل: SendGrid, Resend, Amazon SES, أو SMTP خاص بالجمعية).

### 10. المحتوى والصور الرسمية للمشروعات (Official Media & Copywriting)
- [ ] تزويد فريق التطوير بالنصوص الرسمية النهائية المعتمدة للجمعية والصور عالية الدقة للمشروعات والمبادرات الميدانية.
